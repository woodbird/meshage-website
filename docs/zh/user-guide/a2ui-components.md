# A2UI 交互组件

A2UI（Agent-to-User Interface）是 Meshage 的核心创新之一。AI 代理不仅能发送文字，还能向你投递**原生交互组件**，让人机协作像填表、点按钮一样直观。

## 什么是 A2UI 组件？

传统的 AI 助手只能回复文字或 Markdown。在 Meshage 中，代理可以发送**结构化的 UI 组件**，直接在聊天窗口中渲染为按钮、表单、卡片等原生界面元素。

:::tip 一个例子
你对旅行代理说「帮我订酒店」，代理不是回复一大段文字，而是直接发送一个包含日期选择器、人数输入和价格滑块的表单组件。你只需填写、滑动、点击提交即可。
:::

## 常见组件类型

### 按钮（Button）

代理发送一组按钮供你选择，例如：

- 「同意 / 拒绝」确认按钮
- 「方案 A / 方案 B / 方案 C」多选项按钮
- 「查看详情」链接型按钮

点击按钮后，你的选择会自动发送给代理，无需手动输入。

### 表单（Form）

代理发送一个完整的表单，包含多个输入字段：

| 字段类型 | 说明 |
|----------|------|
| 文本输入 | 单行或多行文本输入框 |
| 下拉选择 | 从预设选项中选择 |
| 单选/多选 | 选择一个或多个选项 |
| 数字输入 | 输入数字，支持步进调节 |

填写完成后点击「提交」，代理会根据你的输入继续处理任务。

### 滑块（Slider）

用于数值范围选择，例如：

- 预算范围：拖动滑块选择 ¥500 ～ ¥3000
- 时间范围：选择会议时长 30 分钟 ～ 2 小时
- 偏好程度：从「不重要」到「非常重要」

### 日期选择器（Date Picker）

代理需要你选择日期或时间时显示，支持：

- 选择单个日期
- 选择日期范围（开始 ～ 结束）
- 选择具体时间（年月日 + 时分）

### 卡片（Card）

信息展示类组件，常用于：

- **搜索结果卡片**：航班、酒店、商品等列表展示
- **确认卡片**：展示订单摘要，包含确认/取消按钮
- **状态卡片**：任务执行进度或结果展示

卡片中可以嵌套图片、文字和操作按钮。

:::info 组件的安全性
A2UI 组件基于白名单机制，代理只能发送平台预先批准的组件类型。所有组件由客户端原生渲染，**不执行任何 HTML 或 JavaScript**，从根本上杜绝了 XSS 等安全风险。
:::

## 如何与组件交互

1. 代理发送的组件会直接出现在聊天消息流中
2. 像使用普通 App 界面一样操作：点击按钮、填写表单、拖动滑块
3. 操作完成后，你的输入/选择会自动回传给代理
4. 代理根据你的交互结果继续处理，可能发送新的组件或文本回复

## 渐进式渲染

复杂的 A2UI 组件支持**流式加载**：

- 代理可以边思考边发送组件片段
- 界面随着 AI 生成的内容实时拼装，无需等待整个组件完成
- 这让交互感觉更加流畅、响应更快

## 降级文本（Fallback）

当组件在某些情况下无法正常渲染时（如旧版客户端或网络异常），系统会自动显示**降级文本**：

- 降级文本以纯文字形式传达组件中的核心信息
- 你仍然可以通过文字回复与代理交互
- 不会丢失任何关键信息

:::warning 组件不显示？
如果你看到「此消息包含富交互组件，请更新 App 以获得最佳体验」的提示，说明你的客户端版本较低。请前往应用商店更新到最新版本。
:::
