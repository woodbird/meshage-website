# 任务审批

Meshage 内置了**人在回路（Human-in-the-loop）**审批机制。当 AI 代理需要执行可能产生重大影响的操作时，必须先获得你的明确批准。

## 为什么需要任务审批？

AI 代理虽然强大，但某些操作一旦执行可能无法撤销或产生重要影响，例如：

- 修改数据库记录
- 发送正式邮件或通知
- 发起支付或转账
- 部署代码到生产环境
- 删除文件或数据

:::tip 安全设计理念
Meshage 的审批机制确保 AI 代理在执行高风险操作前，始终有人类「守门人」把关。这让你既能享受 AI 的效率，又不会失去对关键操作的控制。
:::

## 审批流程

当代理执行到需要审批的操作时，流程如下：

```
代理发起高风险操作 → 触发审批 → 你收到审批卡片 → 审批或拒绝 → 代理继续或停止
```

### 1. 收到审批卡片

代理发送的审批请求会以一张 **A2UI 审批卡片**的形式出现在对话中，包含：

| 信息 | 说明 |
|------|------|
| **操作名称** | 代理即将执行的操作，如「发送邮件」 |
| **操作详情** | 具体要执行的内容，如收件人、邮件正文预览 |
| **风险等级** | 低 / 中 / 高，帮助你判断操作的影响范围 |
| **上下文** | 为什么代理要执行这个操作，前因后果 |

### 2. 审批或拒绝

审批卡片底部有两个按钮：

- **✅ 批准**：同意代理执行该操作
- **❌ 拒绝**：阻止代理执行

:::warning 审批前请仔细阅读
审批卡片中包含了操作的完整信息。请务必仔细阅读操作详情和上下文后再做决定。一旦批准，代理将立即执行该操作。
:::

### 3. 审批后的行为

**批准后：**
- 代理立即执行该操作
- 执行结果会以新消息发送到对话中
- 如果执行成功，显示完成确认
- 如果执行失败，显示错误信息和可能的解决方案

**拒绝后：**
- 代理不会执行该操作
- 代理会发送一条消息，确认操作已被取消
- 代理可能会询问你是否需要调整方案

## 审批超时

如果你在收到审批卡片后长时间未操作：

- 审批卡片会在 **24 小时**后自动过期
- 过期后等同于「拒绝」，操作不会执行
- 代理会收到超时通知

:::info 不要着急
审批没有时间压力（24 小时足够）。如果你不确定是否应该批准，可以先与代理对话了解更多上下文，再回到审批卡片做出决定。
:::

## 审批历史

你可以查看所有审批记录：

1. 进入与代理的对话
2. 点击右上角「**⋮**」→「**审批历史**」

历史记录中展示：

| 字段 | 说明 |
|------|------|
| 操作名称 | 审批的操作内容 |
| 审批结果 | 已批准 / 已拒绝 / 已过期 |
| 审批时间 | 你做出决定的时间 |
| 执行结果 | 批准后的执行状态（成功/失败） |

## 群聊中的审批

在群聊中，当代理发起审批请求时：

- 审批卡片对**所有群成员**可见
- 默认情况下，任何群成员都可以审批
- 群主可以设置仅特定成员有审批权限

## 常见审批场景

| 场景 | 操作示例 | 风险等级 |
|------|----------|----------|
| 发送邮件 | 向客户发送合同确认邮件 | 中 |
| 数据修改 | 更新 CRM 中的客户信息 | 中 |
| 资金操作 | 发起 ¥5,000 的费用报销 | 高 |
| 代码部署 | 将新版本部署到生产环境 | 高 |
| 信息查询 | 查询公开 API 数据 | 低（通常无需审批） |
