# 口令免审批

口令免审批功能让你在保持联系人审批开启的同时，为信任的人提供一条**快速通道**。只要对方在添加你时输入正确的口令，请求就会自动通过。

## 工作原理

```
你设置口令 → 告知信任的朋友 → 对方添加你时输入口令 → 自动通过审批
```

:::tip 使用场景
在线下活动、社群或工作场合中，你可以分享口令给希望快速建立联系的人，免去等待手动审批的时间。活动结束后修改或清空口令即可。
:::

## 设置口令

1. 进入「**我的**」→「**设置**」→「**隐私**」
2. 找到「**口令免审批**」
3. 输入你想设置的口令
4. 点击「**保存**」

**口令规则：**

| 规则 | 要求 |
|------|------|
| 长度 | 4 ～ 32 个字符 |
| 字符 | 支持字母、数字、中文和常见符号 |
| 大小写 | 口令区分大小写 |
| 空格 | 不允许首尾空格 |

## 修改与清空口令

- **修改口令**：在同一设置页面，输入新口令后保存即可
- **清空口令**：清空输入框并保存，口令功能即关闭

修改口令后，旧口令**立即失效**。已通过旧口令添加的联系人关系不受影响。

## 如何分享口令

你可以通过以下方式将口令告知信任的朋友：

- **口头告知**：面对面或通话时告诉对方
- **私信发送**：通过其他渠道（如微信、邮件）发送口令
- **活动海报**：在线下活动中展示口令（注意控制受众范围）

:::warning 安全提示
- **不要**在公开社交媒体上发布口令
- **不要**设置过于简单的口令（如 1234、abcd）
- 定期更换口令，特别是在大范围分享之后
- 口令仅用于联系人请求免审批，不涉及账号登录安全
:::

## 对方如何使用口令

当对方添加你为联系人时：

1. 对方在你的资料页点击「**添加联系人**」
2. 在弹出的请求页面中，找到「**口令**」输入框
3. 输入你分享的口令
4. 点击「**发送请求**」
5. 如果口令正确，请求**自动通过**，双方立即成为联系人

如果口令不正确：

- 请求不会自动通过
- 请求进入你的正常审批队列
- 对方会看到提示「口令不正确，请求已进入审批流程」

:::info 口令与审批的关系
口令是审批的补充，不是替代。即使设置了口令，没有口令或口令错误的请求仍然走正常审批流程。你不会错过任何请求。
:::

## 最佳实践

| 场景 | 建议 |
|------|------|
| 线下活动 | 设置临时口令，活动结束后清空 |
| 团队协作 | 设置团队内部口令，成员入职时告知 |
| 长期使用 | 每月更换一次口令，通知常用联系人 |
| 高安全需求 | 不使用口令功能，所有请求均手动审批 |

## 与关闭审批的区别

| 方式 | 效果 | 安全性 |
|------|------|--------|
| 关闭审批 | 所有人的请求自动通过 | 较低 |
| 口令免审批 | 只有知道口令的人才能自动通过 | 较高 |
| 开启审批（无口令） | 所有请求需手动审批 | 最高 |
