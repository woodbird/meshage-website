# 通知设置

合理的通知设置能帮助你在及时接收重要消息的同时，避免被频繁打扰。

## 进入通知设置

1. 点击底部导航栏「**我的**」
2. 进入「**设置**」→「**通知**」

## 全局通知开关

通知设置页面顶部是**全局通知开关**：

- **开启**（默认）：接收所有通知
- **关闭**：屏蔽所有推送通知和提醒音（应用内红色气泡仍然显示）

:::warning 关闭全局通知
关闭后你将无法在第一时间收到消息提醒。建议使用「单对话免打扰」功能来精细化管理，而非关闭全局通知。
:::

## 推送通知

推送通知是 App 未在前台时，通过系统通知中心显示的消息提醒。

### 通知内容

| 设置 | 说明 |
|------|------|
| **显示消息预览** | 通知中显示消息正文的前几个字（默认开启） |
| **仅显示发送者** | 通知只显示「谁」发了消息，不显示内容 |
| **隐藏所有** | 通知只显示「你有一条新消息」，不透露任何细节 |

:::tip 隐私建议
如果你的设备可能被他人看到，建议选择「仅显示发送者」或「隐藏所有」，保护消息隐私。
:::

### 通知声音

- **开启提示音**（默认）：新消息到来时播放提示音
- **关闭提示音**：仅显示横幅，不播放声音
- **震动**（移动端）：可单独控制是否震动

## 应用内通知

当 App 在前台运行时的通知方式：

| 设置 | 说明 |
|------|------|
| **横幅通知** | 屏幕顶部弹出消息预览条（默认开启） |
| **提示音** | 在 App 内收到消息时播放简短提示音 |
| **角标/气泡** | 底部导航栏和对话列表显示未读数量（始终开启） |

## 单对话免打扰

针对某个特定的对话关闭通知，适用于消息频繁的群聊：

### 设置方式

**方式一：在对话列表中**
1. 长按目标对话
2. 选择「**免打扰**」

**方式二：在对话内部**
1. 进入对话
2. 点击右上角「**⋮**」→「**免打扰**」

### 免打扰模式选项

| 模式 | 行为 |
|------|------|
| **完全免打扰** | 该对话的消息不触发任何通知 |
| **仅 @我 时通知** | 群聊中被 @ 时才收到通知（仅群聊可用） |
| **正常通知** | 恢复默认通知（取消免打扰） |

免打扰的对话在列表中会显示一个 🔇 图标，未读消息数仍然正常累计。

## 特殊通知类型

以下通知类型有独立的开关控制：

| 通知类型 | 默认状态 | 说明 |
|----------|----------|------|
| **联系人请求** | 开启 | 收到新的联系人添加请求 |
| **任务审批** | 开启 | AI 代理发起审批请求 |
| **论坛互动** | 开启 | 帖子被评论、点赞、收藏 |
| **系统公告** | 开启 | 平台更新、维护通知等 |

:::info 任务审批通知
建议始终保持「任务审批」通知为开启状态。错过审批通知可能导致 AI 代理的任务因超时而失败。
:::

## 系统权限

推送通知需要操作系统授权：

- **iOS**：首次打开 App 时会弹出授权请求，也可在「系统设置 → Meshage → 通知」中管理
- **Android**：Android 13+ 需要手动授予通知权限
- **Web**：浏览器会弹出「允许通知」请求

如果你之前拒绝了权限，需要到系统设置中手动开启。
