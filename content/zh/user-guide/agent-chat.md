# 与 AI 对话

在 Meshage 中，与 AI 代理对话就像与人类好友聊天一样简单，但代理能做的事情远不止回复文字。

## 开始对话

1. 在联系人列表中找到已添加的 AI 代理（标有 🤖）
2. 点击进入详情页，点击「**发送消息**」
3. 或者在消息列表中找到与该代理的现有对话

首次与代理对话时，代理通常会发送一条**欢迎消息**，介绍自己的能力和使用方式。

:::tip 第一次聊什么？
不确定怎么开始？试试发送「你能做什么？」或「帮帮我」，代理会告诉你它的能力范围和使用示例。
:::

## 消息格式

与 AI 代理的对话支持多种消息格式：

### 你发送给代理

- **文本消息**：用自然语言描述你的需求
- **文件/图片**：某些代理支持处理文件或图片
- **A2UI 组件交互**：点击代理发送的按钮、填写表单等

### 代理回复给你

代理的回复可能包含以下类型：

| 类型 | 说明 |
|------|------|
| **纯文本** | 普通文字回复，与人类消息体验一致 |
| **富文本** | 包含格式化的文本、列表、链接等 |
| **A2UI 组件** | 按钮、表单、卡片等富交互组件（详见 [A2UI 交互组件](/zh/user-guide/a2ui-components)） |
| **文件/图片** | 代理生成的文件或图片 |
| **任务状态卡片** | 显示当前任务的执行进度 |

## 任务状态

当代理在执行你的请求时，对话中可能会显示**任务状态指示器**：

### working（处理中）

代理正在处理你的请求。你会看到一个加载动画和「处理中...」的提示。

- 复杂任务可能需要几秒到几十秒
- 你可以在等待期间浏览其他对话
- 处理完成后你会收到通知

### input-required（等待输入）

代理需要你提供更多信息才能继续。通常会伴随一个 A2UI 组件，如表单或选择按钮。

:::info 及时回复
当代理进入「等待输入」状态时，你的回复越快，任务完成得越快。如果长时间不回复，部分代理可能会超时。
:::

### completed（已完成）

任务已成功完成。代理会发送最终结果，可能是文本、文件或 A2UI 组件。

### failed（失败）

任务执行失败。代理会说明失败原因，你可以：

- 根据提示修改请求后重试
- 换一种方式描述需求
- 联系代理开发者反馈问题

## 对话上下文

AI 代理会记住**当前对话**中的上下文。你可以连续发送多条消息来细化需求：

```
你：帮我查一下北京到上海的航班
代理：找到以下航班... [显示航班列表组件]
你：只看下午的
代理：已为你筛选下午出发的航班 [更新组件]
你：第二个看起来不错，帮我预订
代理：好的，确认预订信息... [显示确认表单]
```

## 在群聊中与代理互动

在包含 AI 代理的群聊中：

- 使用 **@代理名** 的方式向特定代理发送指令
- 代理只响应被 @到 的消息，不会打扰群聊中的普通对话
- 代理的回复对群内所有成员可见

## 管理与代理的对话

与代理的对话和普通对话一样，支持：

- 置顶、免打扰、归档
- 搜索对话中的历史消息
- 清除对话记录
