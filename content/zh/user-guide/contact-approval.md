# 联系人审批与请求

当你开启了联系人审批（默认开启），其他用户添加你时需要经过你的审批。本页介绍如何管理这些联系人请求。

## 联系人请求流程

完整的联系人请求流程如下：

```
请求方发送请求 → 你收到通知 → 查看请求详情 → 同意或拒绝
```

1. **请求方**在你的资料页点击「添加联系人」，可附带验证消息和口令
2. **你**收到一条推送通知，提示有新的联系人请求
3. 你打开「待审批」列表，查看请求方的资料和验证消息
4. 你选择**同意**或**拒绝**

:::info 口令免审批
如果请求方输入了正确的口令，请求会自动通过，不会出现在你的审批列表中。详见 [口令免审批](/zh/user-guide/passphrase)。
:::

## 查看待审批请求

1. 进入「**联系人**」页面
2. 点击顶部的「**请求**」标签或红色数字气泡
3. 选择「**收到的请求**」

每条待审批请求显示：

| 信息 | 说明 |
|------|------|
| 头像和昵称 | 请求方的基本信息 |
| 账号 ID | 请求方的唯一标识 |
| 验证消息 | 对方发送请求时附带的说明文字 |
| 请求时间 | 请求发送的日期和时间 |

## 审批操作

### 同意请求

点击「**同意**」按钮后：

- 双方立即成为联系人
- 你们可以互相发送消息
- 对方会收到「已同意」的通知
- 请求从待审批列表移至历史记录

### 拒绝请求

点击「**拒绝**」按钮后：

- 请求被拒绝，对方会收到「已拒绝」的通知
- 对方可以再次发送请求
- 请求从待审批列表移至历史记录

:::tip 忽略请求
如果你既不想同意也不想明确拒绝，可以不做任何操作。请求会在 **30 天**后自动过期。
:::

## 我发出的请求

在「请求」页面切换到「**我发出的**」标签，可以查看你发给别人的请求：

| 状态 | 含义 |
|------|------|
| **等待中** | 对方尚未处理你的请求 |
| **已同意** | 对方已通过，你们已成为联系人 |
| **已拒绝** | 对方拒绝了你的请求 |
| **已过期** | 超过 30 天未处理，请求自动失效 |
| **已撤回** | 你主动撤回了该请求 |

## 请求历史

「请求」页面底部有「**历史记录**」入口，记录了所有已处理的请求（包括收到的和发出的），方便你回顾。

历史记录保留最近 **6 个月**的数据。

## 批量处理

当待审批请求较多时，你可以使用批量操作：

1. 在「收到的请求」列表中，点击右上角「**批量处理**」
2. 勾选多个请求
3. 点击「**全部同意**」或「**全部拒绝**」

:::warning 批量操作不可撤销
批量同意或拒绝后无法撤销。如果不确定，建议逐个审批以免误操作。
:::

## 通知设置

你可以在「设置 → 通知 → 联系人请求」中调整请求通知方式：

- **推送通知**：收到请求时弹出系统通知（默认开启）
- **应用内提醒**：联系人标签显示红色数字气泡（始终开启）
